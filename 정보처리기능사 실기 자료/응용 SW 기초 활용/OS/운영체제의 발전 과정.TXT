

	일괄 처리 -> 실시간 처리 -> 다중 프로그래밍 -> 시분할 처리 -> 다중 처리 -> 범용 시스템 -> 분산 처리

	시험 : 일괄 처리 -> 다중 프로그래밍 -> 범용 처리 -> 분산 처리


1.일괄 처리(Batch Processing System)

    처리할 데이터를 한꺼번에 모아 일정한 시간이 경과되거나 일정한 양이 되었을 때 처리하는 방식

    운영체제의 형태 중 시대적으로 가장 먼저 생겨난 형태의 시스템

2.실시간 처리 시스템

    데이터가 발생되는 즉시 처리하는 방식으로, 바로 응답을 받아볼 수 있는 시스템이다. 항상 온라인 상태를 유지해야 한다.

3.다중 프로그래밍 시스템

    한 대의 컴퓨터 메모리에 여러 개의 프로그램을 동시에 적재한 후 실행하는 방식

    CPU와 I/O 장치의 유휴 시간을 줄여 시스템의 사용 효율을 높임

    여러 개의 프로그램이 운영체제의 작업 스케줄링에 의해 시분할적으로 실행됨

4.시분할 처리 시스템

    한 대의 컴퓨터를 동시에 여러 명의 사용자가 대화식의 방식으로 사용하는 시스템

    다중 프로그래밍 방식으로, 모든 사용자들에게 CPU가 일정한 시간대로 나누어서 하나의 작업에게만 자신의 권한을 할당해 주는 방식

    사용자들의 프로그램을 번갈아가며 처리해 줌으로써 각 사용자가 각자 독립된 컴퓨터를 사용하는 느낌을 주는 시스템

    예> 라운드 로빈 스케줄링

5.다중 처리 시스템

    한 대의 컴퓨터에 CPU가 두 개 이상 설치되어 여러 작업을 병행 처리하는 방식

    여러 작업을 동시에 처리하게 함으로써 CPU와 I/O 장치를 최대한 활용하여 시스템의 처리 능력을 향상시킴

6.분산 처리 시스템

    분산된 여러 컴퓨터를 네트워크로 연결하여 처리하는 방식

    데이터가 발생한 지역에서 처리가 가능하므로 중앙 컴퓨터의 부담이 감소함
