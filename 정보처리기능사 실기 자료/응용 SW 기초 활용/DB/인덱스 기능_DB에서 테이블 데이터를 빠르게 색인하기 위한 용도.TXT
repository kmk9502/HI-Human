인덱스 정의

    > DB에서 테이블에 있는 데이터를 빠르게 색인하기 위한 기능을 가짐.

문법

    생성 : CREATE INDEX [인덱스명] ON [테이블]([칼럼명]);

    삭제 : DROP INDEX [인덱스명];

인덱스의 장점

    - 테이블 검색속도와 정렬 속도가 향상

    - 인덱스를 기본키로 자동설정 할 수 있어서 관리가 용이함

인덱스의 단점

    - 인덱스는 별도로 만들어야 해서 파일크기가 늘어남

    - 인덱스에는 갱신 개념이 없어서 추가로 수행해야 함

    - 인덱스를 생성해야 하므로 초기 작업시간이 많이 소요됨
