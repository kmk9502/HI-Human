<SQL 언어> 

    DDL(데이터 정의어)

	- DBA(DB관리자)가 사용함
	- 생성, 변경, 삭제
    ---------------------------------------------------------------------------------
    CREATE	Schema, Domain, Table, View를 생성함

    ALTER	Table에 대한 정의를 변경하는 데 사용함(속성변경)

    DROP	Schema, Domain, Table, View를 삭제함
    ---------------------------------------------------------------------------------

    DML(데이터 조작어)
	
	- 검색, 삽입, 삭제, 변경
    ---------------------------------------------------------------------------------
    SELECT (select 필드 from 테이블명) 		튜플을 검색함 - 정보 열람

    INSERT (insert into 테이블명 Value)		튜플을 삽입함 - Value = 삽입값

    DELETE (delete from 테이블명)		튜플을 삭제함

    UPDATE (update 테이블명 set 변경값)		튜플을 변경함

    * WHERE 명령으로 조건에 해당하는 것만 검색 및 조작할 수 있다. 
    * ex> delete from purchase where item = "apple" 
	  : '주문' 테이블에서 'item' 필드에 있는 "apple"의 튜플을 삭제
    ---------------------------------------------------------------------------------

    DCL(데이터 제어어)
	
	- 데이터의 보안, 무결성, 데이터 회복, 병행 등을 수행하고 DBA가 사용함
	- 저장, 복구, 권한 부여, 권한 회수(취소)
    ---------------------------------------------------------------------------------
    COMMIT	작업이 완료되었음을 확인 <- DB 변경이 완료되면 COMMIT을 눌러 저장

    ROLLBACK	원래의 상태로 복구함 <- 이전 커밋으로 돌아감

    GRANT	사용자에게 권한을 부여함

    REVOKE	사용자에게 권한을 취소함
    ---------------------------------------------------------------------------------

    문제 : 다음의 SQL 명령에서 DISTINCT의 의미를 가장 잘 설명한 것은? 

	Select distinct 학과명 from 학생 where 총점 > 80;

    	1) 학과명이 중복되지 않게 검색한다.
	2) 중복된 학과명만 검색한다.
	3) 동일한 총점을 가진 학생만 검사한다.
	4) 학과명만 제외하고 검색한다.

    답 : 1) 
	 > 학생 테이블에서 총점이 80점 초과인 학생들을 검색하되, 학과명이 중복되지 
	   않게 한다. 즉, 한 학과당 총점이 80점을 초과한 학생 한 명을 검색한다.

    문제 : 제품명과 단가로 이루어진 제품 테이블에서 단가에 대한 내림차순으로 검색하고자
	   한다. () 안에 알맞은 SQL 명령으로 옳게 나열된 것은?

    	SELECT 제품명, 단가 FROM 제품 (1) 단가 (2);

	1) (1)ORDER TO (2) DESC
	2) (1)ORDER BY (2) DESC
 	3) (1)ORDER (2) DOWN
	4) (1)ORDER (2) DESC

    답 : 2)
	 > 정렬을 할 때는 오더바이(ORDER BY), 내림차순은 DESC
	 > 반대로 오름차순은 ORDER BY ASC

    문제 : 다음 SQL 문장의 의미는? (purchase는 구매 테이블, item은 품명이다.)

	select sum(*) from purchase where item = "사과";

	1) 주문한 전체 사과 수량의 평균값을 구한다. <- AVG
	2) 주문한 수량 중 최대값을 구한다. <- MAX
	3) 주문한 수량 중 최소값을 구한다. <- MIN
	4) 주문한 사과의 전체 수량을 구한다. <- SUM

    답 : 4)
	 > purchase 테이블의 전체 필드(*)에서 item(품명) 필드에 있는 "사과"제품의 
	   전체 수량(SUM)을 구한다.
    ---------------------------------------------------------------------------------

    문제 : 다음 SQL 명령문의 의미로 가장 적절한 것은?

	DROP TABLE 학과 CASCADE;

	1) 학과 테이블을 제거하시오.
	2) 학과 필드를 제거하시오.
	3) 학과 테이블과 이 테이블을 참조하는 다른 테이블도 함께 제거하시오.
	4) 학과 테이블이 다른 테이블에 참조 중이면 제거하지 마시오.

    답 : 3)
	 > CASCADE : 제거 대상의 제거와 함께 이를 참조하는 다른 데이터 객체에 대해서도
	   제거 작업을 수행
