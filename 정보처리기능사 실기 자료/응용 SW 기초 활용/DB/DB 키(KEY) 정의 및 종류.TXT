DB KEY 정의

> 테이블에서 다른 데이터와 구분하기 위한 "유일한 값을 가지는 필드" 또는 필드의 집합

> 각각의 튜플을 식별할 수 있는 유일한 성격을 갖는 것으로, 한 테이블에서 적어도 한 개는
  존재해야 한다.
  = 이건 일반적인 속성(필드)의 특징

> 슈퍼키(Super Key) : 한 릴레이션(테이블) 내에 있는 속성들의 집합으로 구성된 키로써
		      모든 튜플에 대해 유일성(Unique)은 만족하지만, 최소성(Minimality)
		      은 만족하지 않음.

> 후보키 : 한 릴레이션에서 유일성과 최소성을 만족하는 키

> 기본키 (Primary Key) : 후보키 중에서 선정되어 사용되는 키로, 절대 NULL이 될 수 
			 없으며, 중복될 수 없다.
			 = 즉, 기본키로 생성된 필드는 NULL이 될 수 없고, 중복을 차단함

> 외래키 (Foreign Key) : 흔히 릴레이션 간의 관계를 표햔할 때 사용하며, 한 릴레이션의
			 a 속성이 참조 릴레이션의 기본키 속성 b가 동일한 도메인 
			 상에서 정의되었을 때 a 속성을 외래키라고 함. 외래키는 
			 참조 릴레이션의 기본키와 동일한 속성을 가져야 함.
			 = 즉, 외래키로 다른 릴레이션의 필드를 가져와 사용할 수 있다.

> 대체키 (Alternative Key) : 후보키 중에서 선정된 기본키 제외한 나머지 후보키


테이블 정의

CREATE TABLE 기본테이블 명 (

	PRIMARY KEY (기본키 속성명),
	UNIQUE (대체키 속성명),
	FOREIGN KEY (외래키 속성명),
	    REFERENCES 참조테이블 (기본키 속성명)
	    [ON DELETE 옵션]
	    [ON UPDATE 옵션]
	[, CONSTRAINT 제약조건명] [CHECK (조건식)]);
